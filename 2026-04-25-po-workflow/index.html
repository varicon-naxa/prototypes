PO Workflow Prototype

Multi-tier sequential approval · pending value changes · soft close · 25 Apr 2026

What's new

This prototype demonstrates a simplified Purchase Order workflow that addresses three problems with the current model: confusing increment/decrement UX on Approve, single-step approval insufficient for civil construction values, and manual closure that lets POs sit forever.

Companion design note: ~/Desktop/bills-test-data/po-workflow-design.md
